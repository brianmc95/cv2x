A 3GPP compliant CV2X Mode 4 Open Source implementation for OMNeT++







Frequently Asked Questions

Can I contribute?

Contributions are most welcome. If you wish to contribute please contact b.mccarthy@cs.ucc.ie with your suggestion. Please fork from the repo and then after development make a pull request on the original.

Where can I get help?

Help is available, though it is important to determine what help you need.

  • OMNeT++ issues should be directed to the OMNeT++ Google Group where people can help with issues with Network simulation
  • Issues with SUMO should be directed to the SUMO mailing list
  • Issues with the model can be directed to the GitHub Issues page and opening an issue
  • Otherwise StackOverflow is always a good resource.
  • Open-CV2X Mode 4 Questions may be directed to b.mccarthy@cs.ucc.ie
Reporting Bugs
  • Issues with the model can be directed to GitHub Issues
  • Issues can also be directed to b.mccarthy@cs.ucc.ie
Common Issues

  • INET: Commonly issues arise due to the incorrect version of INET being used in the project. You specifically need v3.6.6.
  • OSGEarth: When using the project I have noticed issues when OSGEarth is activated. It is necessary to turn this off. This is done using the following command when building OMNeT++. 
    ./configure WITH_OSGEARTH=no WITH_OSG=no
