Open Cellular Vechicle To Everything (C-V2X)

A 3GPP compliant Open Source C-V2X implementation for OMNeT++

Open-CV2X Setup

Requirements GIT Download Alternative Download Make components Cmake setup Run Scenario

Requirements

  • Ubuntu 18.04
  • SUMO 1 or greater
  • CMake 3.13 or greater
  • GNU GCC 7.3
  • OMNeT++ 5.4
  • Boost 1.65.1

Something of note is that these are the versions which have been tested, other versions of the software may still work, though support for such setups will not be possible.

GitHub Download

If you wish to use GitHub then clone the following repository 

git clone https://github.com/brianmc95/artery.git
You then will need to install all the submodules through the following command 
git submodule update --init --recursive
You can also clone the following way to get all at once 
git clone --recurse-submodule https://github.com/brianmc95/artery.git
If you get an error similar to the below use git submodule sync to ensure that the correct versions will install. 
git submodule update --init --recursive error: Server does not allow request for unadvertised object d3a9540affb8a5cc68266fe4ca9500586da125de Fetched in submodule path 'extern/vanetza', but it did not contain d3a9540affb8a5cc68266fe4ca9500586da125de. Direct fetching of that commit failed.

Alternative Download

An alternative means of downloading the model is through the Download section which gives you a tar file with all the necessary files, then simply following on with the next step to setup the model.

Make Components

Now you must make all the necessary submodules this can be done through the make all command in the root directory of the project. This works for all except for veins which require the following to configure and make. 

cd extern/veins & ./configure & make
These can also be built separately by calling make inet, make vanetza and make simulte This can often be preferable as running them in parallel is much quicker (though INET must be made before SimuLte).

Make Components

Now you will need to setup a build directory for the project, as Artery and this project use Cmake as it's build system. 

mkdir build
cd build
cmake ..
Before you can fully build the system you need to ensure that the WITH_SIMULTE option is set in cmake. this is done in the build directory by using cmake-gui or ccmake if that is installed. Now if you run cmake --build . then you will have a fully setup model.

Run Scenario

Now you should be able to run the module using the following command. 

cmake --build build --target run_simulte-cars
This will run the Open-CV2X highway-fast scenario.
